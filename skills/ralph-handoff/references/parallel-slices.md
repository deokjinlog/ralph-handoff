# 병렬 슬라이스 — 여러 루프를 한 번에 돌리기

> SKILL.md 조각 선택·Step 5 의 상세본. 완전 opt-in — **조각을 하나만 넘기면 이 파일은 아무 일도 하지 않는다.**

## 왜 — 지금까지 확인된 명분은 벽시계 시간 하나뿐

**관측 1건** (Wayfare S2 · 2026-07-22 09:51→10:36 · **약 45분** · 13커밋):

- 조각 하나가 **한 시간 안쪽**이었다. 5조각이면 순차로 반나절이 넘는다.
- 그 루프는 **질식하지 않고 완주**했다 — 즉 "컨텍스트가 터져서 쪼갠다" 는 근거는 **아직 없다.**

**주의 — 이건 표본 1이고, 그 조각 자체가 지금 기준으론 잘못 잘린 조각이다.** 그 조각은 **순수 로직 · 화면 없음 · 런타임 의존성 0 · 스캐폴드 미사용**이었다 — 즉 SKILL 이 v0.13.1 에서 ❌ 로 규정한 **수평(층) 슬라이스**다. **여기서 유효한 건 벽시계 수치(45분)뿐이고, 조각의 형태는 본받을 예시가 아니다.** 실제로 그 조각은 아침에 볼 게 없어서 확인용 화면을 사람이 따로 붙여야 했다. DB·결제 조각은 **측정된 적 없다.** 컨텍스트를 병렬의 명분으로 대려면 **터진 기록을 먼저 남겨라.**

## ★ 실측 — "가장 독립적"이라던 조각도 공유 파일을 건드렸다

같은 관측에서 S2 가 실제로 touch 한 파일:

| 자기 것 (안전) | 공유 척추 (머지 충돌) | **루트 산출물 (add/add 충돌 확정)** |
|---|---|---|
| `src/planner/**` | `package.json` · `pnpm-lock.yaml` · `tsconfig.json` · `.gitignore` | `REVIEW.md` · `VERIFY.md` · **`PROMPT.md`** · **`CLAUDE.md`** |

`PROMPT.md` 가 3열에 있는 게 핵심이다 — **병렬이면 완료문구가 슬라이스마다 달라야 하므로 내용이 반드시 다르다.** 루프 2개를 그냥 돌렸으면 lockfile 머지 지옥 + 산출물 4개 덮어쓰기가 확정이었다.

## 규칙 1 — 공유 척추를 **분기 전에** 세운다

빈 프로젝트에서 병렬로 분기하면 각 루프가 자기 `package.json`·lockfile 을 만들어 **반드시 충돌**한다.

**척추** = 패키지 정의 · 락파일 · TS/린트/테스트 설정 · `.gitignore` · **데이터 모델** · 앱 셸 · 라우팅 · 공용 타입.

- 척추는 **베이스 브랜치에서 한 번** 세운다 (사람이 만들거나, 슬라이스 0 으로 순차 1회) → 커밋 → **그 다음에** 워크트리를 갈라라.
- **스캐폴드를 쓰면 분기 *이전* 에 1회만 적용한다.** 워크트리 N개에서 각각 클론하면 그게 바로 규칙 1 위반이다. *(단 Wayfare 관측에선 스캐폴드를 쓰지 않았다 — 척추를 루프가 직접 세웠다. 스캐폴드+병렬 조합은 **미검증**이다.)*
- **테스트 디렉터리도 척추에서 갈라둬라** — 관측의 실제 경로는 평평한 `tests/*.test.ts` 였고, 그대로 두면 다음 슬라이스와 **교집합이 생긴다.** `tests/<슬라이스>/` 로 미리 나눠라.

## 규칙 2 — 산출물을 슬라이스 폴더로 내린다

`REVIEW.md`·`VERIFY.md`·`BLOCKED.md` 를 **루트에 쓰면 루프끼리 덮어쓴다.** 병렬일 때는:

```
docs/features/<날짜>-<슬러그>/
├── <슬러그>-requirements.md   ← 정본 (사람 승인)
├── REVIEW.md · VERIFY.md · BLOCKED.md   ← 이 슬라이스 것
```

**★ 이건 저절로 되지 않는다.** `PROMPT.md` 를 만들 때 **산출물 경로를 이 폴더로 바꿔 박아야** 한다 (SKILL Step 6 의 6-1·6-5·**6-6**·6-8 과 Step 7 의 아침 명령이 기본값으로 루트를 쓴다 — 그쪽엔 `<OUT>/` 자리가 박혀 있으니 **치환만** 하면 된다). 안 박으면 루프는 루트에 쓴다.

그리고 **`PROMPT.md` 와 `CLAUDE.md` 는 머지 대상이 아니다:**
- `PROMPT.md` — 슬라이스마다 다르다. 머지 전에 `git rm` 하거나 슬라이스 폴더로 내려라.
- `CLAUDE.md` — **척추 단계에서 확정**하고 슬라이스는 수정하지 않는다. 수정이 필요하면 `BLOCKED.md`.

## 규칙 3 — 병렬 판정은 **두 축**이다

파일 교집합만 보면 안 된다. SKILL 이 이미 쓰는 두 축을 그대로 쓴다:

| 축 | 통과 조건 |
|---|---|
| **파일 교집합** | 다른 병렬 조각과 **0** (겹치면 순차) |
| **무인 적합도** | 사람이 env 를 꽂아야 green 이 나는 조각은 **병렬 후보가 아니다** |

두 번째 축이 중요하다 — DB·결제·브라우저가 필요한 조각(예: 과금·인증·화면)은 교집합이 0 이어도 **밤새 돌려봐야 BLOCKED 로만 끝난다** (`scaffold.md` 실측: `build`·`dev` 가 Postgres·Stripe env 없이 죽는다). 병렬 슬롯을 거기 쓰지 마라.

**의심스러우면 순차.** 아낀 45분보다 머지 충돌 1건이 비싸다.

## 절차

**1. DAG 를 승인 게이트에서 보여준다** (새 멈춤 만들지 말 것 — 조각 선택 자리에 얹는다. 조각이 하나면 생략):

```
척추(분기 전, 1회) → package.json · tsconfig · 스키마 · 앱 셸 · 가상 DOM 테스트 하네스
   ├─ 일정 짜기 (입력→계획→화면)  교집합 0 · 무인 ★★★ 가상 DOM  → 병렬 ✅
   └─ 공유 링크 (생성→열람 화면)  교집합 0 · 무인 ★★★ 가상 DOM  → 병렬 ✅
        → 구독 과금 (Stripe · 사람 env 필요) → 순차, 사람이 켠 뒤
```

> **★ 조각은 먼저 수직으로 자르고(입력→로직→화면), 그 다음에 겹치는지 본다.** 층으로 잘라 "엔진 조각은 env 가 필요 없으니 병렬" 로 판정하면 SKILL 의 수직 규칙을 어긴다.
> **화면이 있다고 자동으로 무인 부적합이 아니다** — 가상 DOM 으로 검증하면 브라우저 없이 green 이 난다 (실측 2026-07-22: 브라우저 실행은 실패했지만 화면 테스트 17개 green).

**2. 병렬 조각마다 워크트리 + 3파일 + 서로 다른 완료문구:**

```bash
git check-ignore -q .claude/ralph-loop.local.md \
  || printf '.claude/ralph-loop.local.md\n' >> .gitignore   # 추적되면 N브랜치가 매번 충돌한다
git add .gitignore && git commit -q -m "chore: 루프 상태파일 gitignore"

for S in s2-engine s4-search; do
  git worktree add "../$(basename $PWD)-ralph-$S" -b "ralph/$S"
  # 그 슬라이스의 requirements + CLAUDE.md(척추본 복사) + PROMPT.md 배치
  # 완료문구는 슬라이스마다 다르게: S2_ENGINE_DONE / S4_SEARCH_DONE
done
```

> **격리가 되는 이유:** `ralph-loop` 훅이 상태파일을 `.claude/ralph-loop.local.md` **상대경로**로 읽는다 — 즉 **`claude` 를 띄운 디렉터리 기준**으로 갈린다. 반드시 **워크트리 루트에서** 켜라.
>
> **진짜 위험은 "한 창"이 아니라 "같은 폴더"다.** 같은 폴더에서 두 번 켜면 뒤에 켠 게 앞 상태파일을 **덮어써 앞 루프를 하이재킹**한다. 창이 몇 개든 **폴더만 다르면** 안전하다.
>
> 훅의 `session_id` 가드는 `CLAUDE_CODE_SESSION_ID` 가 비면 조용히 꺼진다. 켠 뒤 `grep '^session_id:' .claude/ralph-loop.local.md` 로 값이 있는지 한 번 봐라.

**3. 명령 N개를 출력한다 — 루프는 사람이 켠다.** `ralph-loop` 은 현재 세션에서 도는 대화형 명령이라 슬라이스마다 세션이 필요하다:

```
세션 1:  cd ../<repo>-ralph-s2-engine && claude
         /ralph-loop:ralph-loop "PROMPT.md 를 읽고 그대로 따라라." --completion-promise "S2_ENGINE_DONE" --max-iterations 30

세션 2:  cd ../<repo>-ralph-s4-search && claude
         /ralph-loop:ralph-loop "PROMPT.md 를 읽고 그대로 따라라." --completion-promise "S4_SEARCH_DONE" --max-iterations 30
```

**동시 2~3개를 권한다.** 그 이상은 아침에 사람이 트리아지를 못 한다.

> **★ 2~3 은 추정치다 — 관측 0건.** 이 문서의 다른 수치(45분·충돌 파일 목록)와 달리 **병렬은 아직 한 번도 실행된 적이 없다.** 실제로 돌린 뒤 이 숫자를 갱신해라.

**4. 아침 — 순차 머지 + 트리아지:**

```bash
git merge --no-ff ralph/s2-engine     # VERIFY 가 green 인 것만. 충돌 나면 여기서 사람이 판단
git merge --no-ff ralph/s4-search
git worktree remove ../<repo>-ralph-s2-engine   # 머지해도 워크트리는 자동으로 안 사라진다
```

각 슬라이스 폴더의 `BLOCKED.md`(4열)를 이어붙이면 `TRIAGE.md` 가 된다. 사람의 역할은 **요구사항 승인 + 아침 트리아지** 두 지점이다 — 단 **척추를 미리 세워둔 경우에 한한다.** 척추를 슬라이스 0 루프로 돌리면 그게 끝나야 병렬을 켤 수 있어 **밤중 개입이 한 번 생긴다.** 무인으로 재우려면 **척추는 재우기 전에** 세워라.

## 하지 말 것

| 안티 | 왜 |
|---|---|
| 빈 프로젝트에서 병렬 분기 | 각자 `package.json`·lockfile 생성 → **반드시 충돌.** 척추 먼저 |
| 워크트리마다 스캐폴드 클론 | 그게 바로 위 충돌이다. 스캐폴드는 **분기 전 1회** |
| `REVIEW/VERIFY/BLOCKED` 를 루트에 | 루프끼리 덮어쓴다. `PROMPT.md` 에 슬라이스 폴더 경로를 **박아야** 바뀐다 |
| `PROMPT.md`·`CLAUDE.md` 를 머지 대상으로 | 내용이 슬라이스마다 달라 add/add 충돌. 머지 전 정리 |
| 파일 교집합 있는 조각을 병렬 | 머지 지옥. 의심스러우면 순차 |
| 사람 env 필요한 조각을 병렬 슬롯에 | 밤새 돌려도 BLOCKED 로만 끝난다. 무인 적합도도 같이 봐라 |
| 같은 폴더에서 루프를 두 번 켬 | 뒤가 앞 상태파일을 **덮어써 하이재킹**한다 |
| 루프가 도는 폴더를 사람·다른 에이전트가 동시에 편집·빌드 | **실제로 터졌다(academy 벤치마크).** 커밋이 뒤섞이고 상태 판정이 어긋난다. 멈춘 뒤에 보거나 사본에서 봐라 |
| 완료문구를 슬라이스마다 같게 | 훅이 정확 일치로 본다. 슬러그마다 다른 ASCII |
| 동시 4개 이상 | 아침 트리아지가 안 된다. 2~3개 |
| 컨텍스트가 터져서 병렬한다고 말함 | **관측 1건에선 질식이 없었다.** 지금 확인된 명분은 벽시계뿐 — 대려면 터진 기록을 먼저 남겨라 |
| 조각을 **층으로** 자른 뒤 병렬 판정 | 수직 규칙 위반. 먼저 기능으로 자르고(입력→로직→화면) 그 다음 교집합을 봐라 |
| 화면 조각이라서 무인 부적합으로 단정 | **가상 DOM 이면 브라우저 없이 green 이 난다.** 사람 env 가 진짜 필요한 건 DB·결제·외부 키다 |
