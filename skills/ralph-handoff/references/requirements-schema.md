# 요구사항 스키마 — 기획서 없이 인터뷰로 채우는 슬롯

> SKILL.md Step 2 의 no-doc 인테이크 상세본. **자유 문답이 아니라 빈 슬롯 채우기.**
> 원칙은 이 레포 그대로: **예고 · 예산 · 기본값** (Step 0 과 동일). 산출물은 Step 3 요구사항 형식과 1:1.

---

## 채울 슬롯 — 이걸 다 채우면 요구사항이 완성된다

| 슬롯 | 최소 | 질문 힌트 | 비어 있으면 |
|---|---|---|---|
| **personas** | 1 | 누가 쓰나? 결제 주체 ≠ 사용 주체? | 화면·권한이 붕 뜬다 |
| **core_journeys** | 3 | "돈 되는" 저니만. 가입~첫 가치 도달 포함 | 만들 순서를 못 정한다 |
| **features** | — | 각 저니를 실현하는 기능. **각각 AC + `[unit]`/`[e2e]` 태그** | 검증 배선이 안 된다 |
| **non_functional** | — | 동시접속 · 데이터 보존 · 권한 모델 · 규제 | NFR 0 → 루프가 무한정 만든다 |
| **out_of_scope** | 3 | 안 만들 것 | 루프의 울타리가 없다 |
| **data_model_draft** *(선택)* | — | 엔티티·관계 (기술 중립) | 병렬 슬라이스가 스키마를 두고 충돌 (증분4) |

`features → AC → [unit]/[e2e]` 가 곧 증분1의 브라우저 검증(6-8)으로 흘러간다. **인테이크와 검증이 한 줄로 이어진다.**

> **`data_model_draft` 는 선택** — Step 3 요구사항 형식엔 아직 데이터모델 섹션이 없다. 증분4(병렬 슬라이스) 전까진 `CLAUDE.md` 의 폴더구조·설계 원칙에 녹여도 된다.

## 인터뷰 규칙

1. **예고 먼저** (Step 0 형식): *"빈 슬롯 N개입니다. 최대 K문(기본 3~5) 안에 채우고, 나머지는 기본값+근거로 제안합니다. 그다음 요구사항+CLAUDE.md 승인 1번."*
2. **질문 1개 = 슬롯 1개 이상**을 닫아야 한다. 우선순위: **persona → 핵심 저니 1개 → 만들거나-부수는 제약 하나.** 가장 추측을 줄이는 것부터.
3. **닫지 못하는 흥미 질문 금지.** *"이런 것도 있으면 좋죠?"* 류 확장 질문은 묻지 말고 `out_of_scope` 로 밀어라.
4. **예산 소진 → 기본값.** 빈 슬롯을 "합리적 기본값 + 근거 1줄" 로 채우고 `[기본값]` 표시. 승인 게이트에서 한 번에 보여준다.
5. **암시된 슬롯은 추론**, `[추론]` 표시. 다시 묻지 마라.
6. **AC 에는 반드시 `[unit]`/`[e2e]` 태그** (증분1). 저니형·화면 → `[e2e]`, 순수 로직 → `[unit]`.

## 기본값 정책 (defaults) — 지어내지 말고, 흔하고 되돌리기 쉬운 쪽

정보가 없을 때 **가장 흔한 · 가장 되돌리기 쉬운** 선택을 기본값으로 두고, 근거를 붙여 사람이 한눈에 뒤집게 한다.

| 미정 슬롯 | 기본값 | 근거 표기 |
|---|---|---|
| 권한 모델 | 단일 역할(로그인=전권) | `[기본값: 세분 RBAC 는 out_of_scope]` |
| 데이터 보존 | 소프트 삭제 없음, 하드 삭제 | `[기본값: 규제 미확인 — 보존은 나중에]` |
| 동시접속 | 동시 수십 명 규모 | `[기본값: 초기 트래픽 가정]` |

각 기본값에 `[기본값: <근거>]` 를 붙인다. **승인 게이트에서 사람이 한 마디로 뒤집을 수 있어야** 한다.

## no-doc 는 조각 선택(멈춤 ①)이 없다

기획서가 없으면 자를 조각이 없다. 그래서 흐름은:

```
예고 → K문 인터뷰 → 스키마 채움(+기본값) → 요구사항 + CLAUDE.md → 승인(멈춤 ②) 1번
     → (커밋 · 워크트리 · PROMPT.md · 실행 명령)
```

**스택(그린필드):** 요구사항은 기술 중립이라 스택은 슬롯이 아니지만, 기존 코드가 없으면 스택을 모른다. **스택도 같은 인터뷰 예산 안에서 1문** 물어 `CLAUDE.md` 로 보낸다 — Step 4 가 **따로 멈춰 묻지 않도록.** 승인 멈춤은 그래도 1번, 질문만 예산 안에서 하나 는다.

**멈춤은 승인 1번뿐이다.** 인터뷰 질문은 예고된 예산 안에서만. 이것이 "질문 두 번만" 을 깨지 않는 이유 — 기획서 있을 때의 2번(조각+승인)과 **다른 레짐**일 뿐, 원칙(예고·예산·기본값)은 동일하다.

## 하지 말 것

| 안티 | 왜 |
|---|---|
| 한 줄 답으로 요구사항 통째 지어냄 | 슬롯이 비어 루프가 추측으로 메운다 (README 가 인정한 실패) |
| 예산·예고 없이 질문 | Step 0 예고가 거짓말이 된다 |
| `out_of_scope` 를 비워둠 | 루프가 울타리 없이 무한 확장한다 |
| AC 에 `[unit]`/`[e2e]` 태그 안 닮 | 검증기 배선 불가 (증분1) |
| `brainstorming` 에 위임 (명시 요청 없이) | 게이트가 5개 더 생긴다. 스키마로 직접 채워라 |
| 인터뷰가 8문 넘게 늘어짐 | 이 레포는 질문을 혐오한다. 3~5문 + 기본값으로 끝내라 |
