> ## 무인 루프는 채팅을 못 봅니다.
>
> **파일에 없으면 없는 겁니다.**
> 이 플러그인은 루프에 필요한 **파일 3개**를 갖춰 격리해주고, 명령 한 줄을 출력합니다.
> **루프는 당신이 겁니다.**

---

## 루프가 받아야 하는 건 3개

| 파일 | 답하는 질문 | 없으면 |
|---|---|---|
| **`CLAUDE.md`** | **뭘로** 만드나 (스택 · 구조) | **루프가 스택을 지어냅니다** |
| **`<slug>-requirements.md`** | **뭘** 만드나 (기술 중립) | 만들 게 없습니다 |
| **`PROMPT.md`** | 루프가 **어떻게 도나** | 무한 루프 / 거짓 완료 |

요구사항은 **일부러 기술 중립**이라 그것만 넘기면 스택 키워드가 **0건**입니다 — 루프가 Flask 든 Express 든 맘대로 골라요. **스택은 원래 `CLAUDE.md` 자리**인데 그 자리가 비어 있는 게 문제였습니다. 이 플러그인의 존재 이유가 그겁니다.

---

## 실행 — 설치부터 루프까지

**① 설치 (한 번)**

```
/plugin marketplace add anthropics/claude-plugins-official
/plugin install ralph-loop@claude-plugins-official
/plugin marketplace add deokjinlog/ralph-handoff
/plugin install ralph-handoff@ralph-handoff
```

세션을 재시작합니다.

**② 프로젝트 폴더 (한 번)**

```bash
mkdir -p ~/projects/내프로젝트/docs && cd ~/projects/내프로젝트
git init
printf 'docs/*.pptx\ndocs/*.pdf\ndocs/*.hwp\ndocs/*.xlsx\n' > .gitignore   # ★ 기획서를 옮기기 전에
mv ~/어디/기획서.pptx docs/          # 있으면. 없어도 됩니다
git add .gitignore && git commit -m "chore: 초기"
claude
```

> **기획서는 프로젝트 폴더 안에** 두세요 (플러그인 폴더가 아니라 — 설치된 플러그인은 캐시로 복사돼서 아무도 못 읽습니다).
> **`.gitignore` 를 기획서보다 먼저 만드는 게 핵심입니다** — 한 번 커밋되면 `.gitignore` 로는 못 뺍니다(추적 중인 파일엔 효력이 없어요). 스킬도 **추적 여부까지 직접 검사**해서, 이미 추적 중이면 `.gitignore` 추가로 끝내지 않고 `git rm --cached` 까지 합니다.

**③ 발동 — `/ralph-handoff`** (또는 그냥 "랄프한테 넘겨줘")

- **기획서가 있으면** → 읽고, 크면 어느 조각을 넘길지 물어봅니다 ← **멈춤 ①**
- **기획서가 없으면 → 스키마 인테이크** (소크라테스식이되 **자유 대화가 아니라 빈 슬롯 채우기**). 먼저 예고하고 3~5문으로 채웁니다:

```
ℹ️ 기획 자료가 없네요 → 스키마 인테이크.
   빈 슬롯 6개를 최대 3~5문으로 채우고 나머지는 기본값 → 승인 1번.
   ① 누가 쓰나요? (결제 주체 ≠ 사용 주체?)
   ② 돈 되는 핵심 저니 하나
   ③ 만들거나-부수는 제약 하나
```

당신 답으로 슬롯(persona · 저니 · 기능+AC · NFR · 범위밖 · 데이터모델)이 채워지고, **안 물은 슬롯은 "기본값 + 근거"** 로 채워집니다.

**④ 승인 ← 멈춤 ② (유일한 방어선)** — 요구사항과 `CLAUDE.md` 를 **한 메시지에** 보여줍니다. 루프가 몇 시간 도는데 방향이 틀리면 전부 버리니까요.

**⑤ 자동 — 여기서부턴 안 묻습니다** — 커밋 → 워크트리(지킬 코드가 있을 때만, 빈 프로젝트면 브랜치) → `PROMPT.md` → **실행 명령 출력**

**⑥ 루프 켜기 — 당신이**

```
cd <경로> && claude
/ralph-loop:ralph-loop "PROMPT.md 를 읽고 그대로 따라라." --completion-promise "<완료문구>" --max-iterations 30
```

**⑦ 아침에**

```bash
cat VERIFY.md    # 검증을 진짜 돌렸나 (없으면 안 돌린 것)
cat REVIEW.md    # 검문자가 뭘 잡았나
cat BLOCKED.md   # 막힌 것 · 사람이 볼 것 (4열 트리아지)
git log --oneline
```

> **루프가 아직 돌고 있으면 그 폴더를 건드리지 마세요.** 멈춘 뒤에, 또는 다른 브랜치를 체크아웃한 사본에서 보세요 — 동시에 같은 파일을 쓰면 레이스가 납니다.

> **멈춤은 여전히 최대 두 번입니다** — no-doc 엔 조각 선택이 없어 **인터뷰(예고된 3~5문) + 승인 1번**으로 끝납니다.

## `PROMPT.md` 에 뭐가 들어가나 — 대부분 실제로 터진 것들

| 장치 | 뭘 막나 |
|---|---|
| **상태 판정 — 매 반복 첫 동작** | 루프는 **매 반복 기억이 없습니다.** 진척을 파일에서 읽게 하지 않으면 같은 일을 반복하거나 다 됐다고 착각합니다 |
| **묻지 말고 `BLOCKED.md`** | 새벽 3시엔 답할 사람이 없습니다. 질문하면 **무한 루프** |
| **한 반복 = 한 단계** | 한 번에 다 하려다 **컨텍스트가 터집니다** |
| **완료 조건에 사람이 센 숫자 금지** | 계획서엔 `22건`, 실제는 `25건` 이었습니다. **기계끼리 대조**시킵니다 |
| **검문 관문** | **자기 답안지로 자기 채점.** 다른 목적의 서브에이전트가 계획서만 가로질러 읽습니다 |
| **`VERIFY.md` 증거 강제** | 훅이 상태파일을 지우는 지점이 **9군데** — 그중 **정상 종료는 2개(promise 감지·최대반복)뿐**이고 나머지 7개(transcript 파싱 실패·상태파일 손상)는 **조용히 죽습니다.** 막을 수 없으니 **터미널 출력을 남기게** |
| **`<promise>` 태그 필수** | 태그 없이 문구만 쓰면 **루프가 영원히 안 끝납니다** ↓ |
| **브라우저 검증 (있을 때)** | `[e2e]` AC 를 Playwright 로 실제 클릭. 브라우저 없으면 BLOCKED. **무인 실행 사고 기록이 아니라 스모크 실측 기반** |
| **취소 금지** | 완료 안 됐는데 빠져나가는 문을 막습니다 |

**`<promise>` 태그가 왜 필수인가** — 훅의 `perl … <promise>(.*?)</promise> …` 는 **매치가 없으면 원본을 그대로 둡니다.** 태그가 없으면 비교 대상이 **메시지 전체**가 되는데, **LLM 은 거의 항상 앞에 한 줄 붙입니다.** → 매치 실패 → `--max-iterations` 를 다 돌고 죽어요. 그래서 `PROMPT.md` 에 **태그째** 박습니다.

**왜 "번역" 이 필요한가** — 게이트 워크플로는 *"막히면 멈추고 물어봐"*, 루프는 *"막히면 계속 시도"*. **정반대**입니다. 그래서 모든 게이트를 *"`BLOCKED.md` 에 적고 건너뛴다"* 로 번역합니다.

## 실제로 돌려본 결과

발표자료 30장 → 조각 선택 → 요구사항 → `CLAUDE.md` → `PROMPT.md` → 루프. **검문 관문**이 반복 3에서 남긴 `REVIEW.md` 머리말입니다:

> **검문자**: 별도 서브에이전트 — 계획서·요구사항·설계만 읽음. **코드 미열람.**
> **원칙**: 계획서를 쓴 주체가 채점하지 않는다. **다른 프롬프트 · 다른 컨텍스트 · 다른 목적.**

그리고 **숫자 게이트가 기계끼리 대조**됩니다 — 사람이 센 숫자가 하나도 없습니다:

| 검사 | 결과 |
|---|---|
| 테스트 수 (계획서 정의 vs 수집) | **21 == 21** |
| AC 수 (요구사항 vs 계획서) | **20 == 20** |
| 의존성 (허용 목록 밖) | **0** |
| 범위 (요구사항 §5 침범) | **0** |

루프가 **스스로 모순을 잡고 고치기도** 합니다 — `57bda92 docs: 구현계획서 + 개발방향 모순 2건 수정 (ralph 반복 2)`. **코드에서 우회하지 않고 문서를 고칩니다.**

## 상황이 맞을 때만 켜지는 것들

**AC 에 `[unit]` / `[e2e]` 태그를 답니다.** 수용 기준마다 태그가 붙고, 그게 **어느 검증기가 도는지**를 정합니다 — `[unit]`은 테스트 러너, `[e2e]`는 진짜 브라우저. 브라우저를 못 띄우면 `[e2e]`는 `BLOCKED.md` 로 내려가고 루프는 계속 갑니다. **태그가 없으면 검증 배선이 안 됩니다.**

**골든 스캐폴드 (웹 SaaS 그린필드일 때만).** 인증·구독 과금·멀티테넌시 배관을 루프가 매번 새로 짜면 **매번 다르게 틀립니다.** 그래서 검증된 스타터를 **묻지 않고 초안에 넣습니다** (승인 게이트에서 빼시면 됩니다). 배관은 읽기 전용이 되고 손대려 하면 `BLOCKED.md` 로 갑니다. **실측 경계**: 자율로 나오는 green 은 `install` + 타입체크까지고, `build`·`dev` 는 DB·결제 env 를 요구해 죽습니다 — 그건 사람 몫으로 넘어갑니다. 웹 SaaS 가 아니거나 기존 코드가 있으면 **아무 일도 안 합니다.**

**병렬 슬라이스 (조각이 둘 이상일 때만).** 안 겹치는 조각을 워크트리로 갈라 동시에 돌립니다. 조각 하나면 아무 일도 안 합니다. 전제가 셋이라 **의심스러우면 순차**가 기본입니다 — ①공유 척추(패키지·락파일·스키마)를 **분기 전에** 세울 것 ②산출물을 조각 폴더로 내릴 것 ③`PROMPT.md`·`CLAUDE.md` 는 머지 대상이 아님. **동시 2~3개는 아직 추정치입니다** (병렬은 실행 관측 0건).

## 뭘 잃는지 · 언제 안 맞는지

| 잃는 것 | 뜻 |
|---|---|
| **설계의 기술 선택권** | 루프는 *"대안 비교 → 추천 자동 선택"* 입니다. 고르는 것도 자기가 합니다 |
| **계획을 검토할 사람** | 루프가 쓴 계획을 루프가 실행합니다. **검문 관문**을 넣었지만 완벽하진 않아요 — 아침에 개발방향을 꼭 읽어보세요 |

**안 맞는 경우** — 기술 선택이 중요할 때 · 요구사항이 모호할 때(루프가 추측으로 메웁니다) · **되돌리기 어려운 작업**(DB 마이그레이션 · 결제 실연동 · 배포)이 포함될 때. 마지막은 전부 `BLOCKED.md` 로 갑니다. **사람이 하세요.**

**규모의 현실** — AC 가 수십 개인 큰 앱은 한 번의 무인 실행으로 안 끝납니다(한 반복=한 단계라 처리량이 벽). 벤치마크 관측: 랄프는 **설계 단계(백트래킹·탐색상한·독립검사기까지)는 완주**하지만 코드는 완주 못 합니다. 큰 건 **화면 하나=한 조각**으로 잘게·수직으로 자르고, AC 채점표의 **부분 점수**를 정상으로 보세요.

**조각은 층이 아니라 기능으로 — 수직으로 자릅니다.** 한 조각이 *입력 → 로직 → 화면*을 관통해야 합니다. "엔진 층 / 화면 층" 으로 자르면 로직 조각은 **정의상 아침에 볼 게 없습니다** (실제로 터졌습니다 — 테스트 16개 green, 볼 것 0).

**그리고 두 축을 봅니다** — *무인 적합도*(루프가 인프라 없이 검증 가능한가)와 *눈으로 확인*(아침에 볼 게 있나). 둘은 자주 반대입니다: 순수 로직은 잘 검증되지만 볼 게 없고, 화면은 눈에 보이지만 **브라우저가 있어야** 루프가 검증합니다. 볼 게 없는 조각이면 **확인용 최소 화면을 묻지 않고 요구사항에 넣습니다** (승인 게이트에서 빼실 수 있게). *"화면은 다음 조각이 담당한다"* 는 **면제 사유가 아닙니다.**

---

## 이 플러그인이 안 하는 것

- **루프를 시작하지 않습니다.** 명령을 출력만 해요 — 사람이 겁니다
- **현재 브랜치를 안 건드립니다.** 전부 워크트리/브랜치 안에서
- **승인 없이 아무것도 안 지웁니다**

## 같이 쓰면 좋은 것

[**intent-locked-workflow**](https://github.com/deokjinlog/intent-locked-workflow) — 기획서를 요구사항으로 만드는 4단계 게이트 워크플로. **필수는 아닙니다** (없으면 이 스킬이 직접 씁니다).
